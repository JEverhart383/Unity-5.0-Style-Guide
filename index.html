Buttons

All buttons for VFB extend Bootstrap 3 default's button class and styles. Therefore, you must apply the .btn base class to any of the button variations below for padding, margin, etc. and to override vendor default styles.

Divs can be styled using all button classes, but active states will only be applied to buttons using the either a, button type="button", or input type="button"

**Do not use button type="submit" or button without a type, which defaults to submit. DNN does not play well with form submissions.

Example of Toggle Buttons

Toggle radio buttons are constructed using a number of other conventions used with buttons elsewhere in the style guide. As with components defined in Input Groups, to avoid using JS to style :checked radio buttons, we use the adjacent sibling selector. Thus, the structure of the markup is very important. For these components, the actual radio inputs themselves are hidden using the .vfb-hidden-tick class and the labels are styled accordingly.



	
                            <div class='vfb-center vfb-toggle-buttons'>
                                <input type='radio' class='vfb-hidden-tick' id="toggleOption1" name="radio-group"/>
                                <label class='vfb-toggle-btn btn button-sm' for="toggleOption1">
                                	<span class='tick-text'>Option 1</span>
                                </label>
                                <input type='radio' class='vfb-hidden-tick' id='toggleOption2' name="radio-group"/>
                                <label class='vfb-toggle-btn btn button-sm' for='toggleOption2'>
                                	<span class='tick-text' data-bind='text: value'>Option 2</span>
                                </label> 
                            </div>

Inline CTAs

All inline CTAs need to be in a div class="container-fluid" and placed in one of the valid full width containers in the skin. Most of the styling is done with color utility classes, and the layouts for text are constructed using the Bootstrap grid.

Inputs contained within a respective .vfb-cta-* class will all receive default stylings as a descendant.


CTA Brief

The CTA Brief component is constructed with a div class="vfb-cta-brief vfb-color-secondary vfb-bg-footer" to contain the grid layout, text, and input. At screen widths less that 768px, the CTA Brief collapses into a 12-col layout. The height of the CTA Brief at widths higher than 768px is 100px but will expand to fit its containing elements as the layout collapses.

CTA Verbose

Input Groups

For both the checkbox and radio input groups, there are a few things to be aware of when implementing these components. To avoid using JS to style elements based on :checked property and to get around the need to override vendor specific styles on input elements, we've used a tricky but well-know front-end dev technique to style labels.

First, we hide all of the inputs off-screen and have users interact directly with each input's corresponding label. To hide any input off-screen, you can apply the .vfb-hidden-tick class. Next, you need make sure each combination of input/label has correctly used id and for attributes that tie them together.

Important Notes About Markup Structure

Lastly, to use the :checked property of each input to style its corresponding label without JS, we use the adjacent sibling CSS selector. You can see an example of this below:

        	input.vfb-hidden-tick:checked + label.vfb-tick {
    			/*Styles would go here to modify label when input checked*/
			}

Therefore, the structure of the actual markup is very important here since the adjacent sibling selector will only look at the element immediately following the input. In other words, when creating or dynamically generating these elements, the input needs to come first, followed immediately by its corresponding label. See a simplified example of this markup below:

        <input type='checkbox' class='vfb-hidden-tick' id="option1"/>
                <label class='vfb-tick' for="option1"></label>
